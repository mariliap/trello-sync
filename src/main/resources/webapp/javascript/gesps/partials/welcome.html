Check remember me to create a browser session ( sessionStorage ) that stores the credentials.
When a page is reloaded, credentials retrieved from the session are used to automatically login to reload authorizations and to stay on the same page.

Open a web console, refresh the page and check the logs.

Filter anon is applied to the path /login in accordance with angular-shiro default configuration. 

urls : {
	'/' : 'anon',
	'/index' : 'anon',
	'/login' : 'anon',
	'/signin' : 'anon',
	'/logout' : 'logout',
	'/signout' : 'logout'
}

Try to type in the following path /#/app in your browser navigation bar.

In the logs you will notice that the authc filter is applied and as the current Subject is not autenticated he is redirected to the configured loginUrl which is by default /login .
